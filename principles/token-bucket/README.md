# 令牌桶的突发与平均速率约束

## 定义

令牌桶首先是一个关于许可守恒的离散状态模型，而不是某个中间件命令。设桶容量为 B，当前可用量为 A，补充间隔为 R；每经过一个完整间隔最多产生一个令牌，A 永远不能超过 B。一次请求需要消耗给定数量的令牌：余额足够时原子扣减并放行，余额不足时拒绝且不透支。容量允许空闲期积攒有限许可，因此系统可以接受短时突发；固定补充速度又限定了持续流量。

[SOURCED:token-bucket-standard-definition:source-rfc-3290-token-bucket] [RFC 3290 Appendix A](https://www.rfc-editor.org/rfc/rfc3290.html#appendix-A) 用容量与速率描述令牌桶，并说明符合条件的流量可在桶所允许的突发范围内发送。这里仅转述这个抽象定义；仓库实现进一步把时间、错误和整数边界写成可执行契约。

## 不变量

核心不变量是零不小于可用量的下界与容量上界同时成立，即任何可见状态都满足零到 B 的闭区间。放行 amount 后余额恰好减少 amount；拒绝不会减少余额，但会提交合法的时间观察，因为时间流逝本身可能产生令牌。输入数量为零或大于容量不是一次正常拒绝，而是调用契约错误，必须在读取时间之前返回，防止无效请求偷偷推进状态。

时间也有独立不变量。每个实例保存补充锚点和最近观察时刻：锚点只按完整间隔向前移动，余下的不足一个间隔部分必须保留；最近观察时刻用于发现同一间隔内的回退。桶即使已经满了，经过的完整间隔也必须推动锚点，否则随后一次大量消费会错误地复用历史上本应丢弃的令牌。所有读写共享同一互斥边界，使“补充、判断、扣减”成为一个不可分割的状态转换。

最后，等待时间也是状态契约的一部分。余额不足时，返回值必须覆盖所有缺少的令牌，而不只是等到下一个令牌；如果数学等待时间超出可表示时长，则结果饱和到最大时长而不能整数回绕。这样调用者看到的是保守且单调的重试建议，不会因为极值输入获得一个虚假的短等待。

## 推导

从守恒关系出发，任意时间窗口内能够放行的总量，不可能超过窗口开始时已有的余额加上窗口内完整补充间隔产生的令牌。初始余额最多为 B，因此空闲后第一轮最多承受 B 个单位的突发；当这些余额耗尽，长期新增许可的斜率由 R 唯一决定。容量改变瞬时缓冲，补充间隔改变平均斜率，两者是不同旋钮，不能用增加容量来永久提高吞吐。

当请求到达补充边界之前一纳秒，完整间隔数仍为零，所以即使时间非常接近边界也必须拒绝；到达边界时才产生一个完整令牌。若一次请求缺少 N 个令牌，等待由当前间隔剩余时间加上后续 N 减一段完整间隔组成。这个推导解释了为什么只返回“距下一次补充”的实现会对批量请求过早重试，也解释了为何余数必须归属于锚点而不能在每次读取后清零。

并发情形没有新的许可来源。多个调用在同一逻辑时刻竞争时，具体哪个调用获胜可以由调度决定，但放行总量和最终余额必须确定；测试因此比较总数与余额多重集，而不依赖 goroutine 的赢家顺序。这把正确性从偶然执行顺序中分离出来，也为后续按租户、节点或全局协调扩展提供稳定基础。

## 失败边界

第一个失败边界是时间。若系统允许时钟回退后继续计算，已消费的许可可能被重复生成；只比较补充锚点还不够，因为回退可能发生在同一个间隔内，所以必须另存最近观察时刻。另一个边界是超长向前跳跃：标准时长类型只有有限范围，静默采用饱和值再继续补充会把不可表达的事实伪装成正常状态，因此实现返回明确的时间溢出错误且保持原状态。

第二个边界是整数极值。容量和余额使用无符号整数时，直接执行“余额加完整间隔数”可能先回绕再取最小值；安全顺序是先比较可用空间，达到或超过空间就直接设为容量。重试时间的乘法同样先检查剩余表示空间，超界时饱和。错误路径也必须保持原子性：非法 amount、回退和时间溢出均不得改写余额、锚点或最近观察时刻。

令牌桶只描述单个一致状态机的许可边界，并不自动解决分布式所有权。多个节点各持一个桶会把总容量与补充速率按副本数放大；共享协调器故障时，fail-open 与 fail-closed 又分别牺牲配额精度或可用性。公平排队、租户权重、热点隔离和持久恢复也不由这个原语单独保证，必须由上层协议另行定义。

## 可复用实验

[MEASURED:token-bucket-bounds-burst-and-average-rate] `burst-and-refill-boundary` 实验在固定 UTC 纪元、容量四、每秒补充一个令牌的条件下运行参考模型与互斥实现。轨迹先一次消费全部容量，再立即请求一个单位；随后在补充边界前一纳秒和边界时分别探测，并在三秒处一次消费两个单位，以确保实现不仅会处理第一次补充，也能累计多个完整间隔。

实验记录请求总数、放行与拒绝数、容量、最终余额、观察到的最大余额、四个边界探针以及参考模型不一致数。所有正确性指标都是整数，事件使用逻辑时间，seed 固定；断言要求突发有界、边界前拒绝、边界时只补一个、容量从不超界并且每个决定与独立模型一致。该小实验验证状态机，不代表生产网络延迟、跨节点一致性或真实容量上限。

## 关联题目

令牌桶可复用于 API 网关、任务入口、消息消费、租户配额和过载保护，但题目重点通常在“谁拥有这份余额”以及故障时选择哪种语义。`distributed-rate-limiter` 会把同一原语放进共享协调与每节点独立两种所有权模型，比较全局配额偏差，并显式讨论协调器不可用时的放行策略。其他题目也可以复用该实验来验证局部限速，却不能直接把局部通过外推成全局正确。
