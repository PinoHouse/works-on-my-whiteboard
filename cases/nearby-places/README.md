# 附近地点

## 表面题目

给定用户位置、半径和属性过滤，返回附近且当前合格的地点。成功不是 cell 相同就算附近，而是候选经过精确距离和营业、可见、租户等当前资格校验，并对移动对象的位置陈旧给出明确边界。题名容易混合静态 POI 与实时司机：前者位置很少变但营业状态变化，后者位置持续变化且旧坐标迅速失去意义。

## 反问与边界

先问对象是静态店铺、活动还是移动实体；查询按圆形半径、可达时间还是行政区，是否需要固定 K、排序和分页。确认位置更新频率、允许 age、搜索 p99、最大半径、密集城市与稀疏地区分布、过滤选择率、地球曲率精度及跨国数据驻留。召回目标应在一套带真值的业务样本上校准，不能凭空声称百分比。本题不负责导航路径，也不把用户上报坐标无条件当可信。

权威地点记录拥有坐标版本、营业和可见状态；空间 cell、属性倒排和边缘缓存是派生索引。每个候选携带 `place_version,index_generation,location_observed_at`，最终读当前资格。刷新更快会放大移动对象写入、cell 迁移和缓存失效。

## 客观模型

接口为 `UpsertPlace(id,version,location,attributes)`、`SearchNearby(point,radius,filters,K,snapshot)`。空间索引把点映射到层级 cell；查询覆盖与圆相交的 cell，取候选后以 Haversine 或局部投影算精确距离。候选量约 `C ≈ density * covered_area / filter_selectivity`，城市密度增加十倍会直接增加过滤 CPU。

距离资格为 `exact_distance(p,q) ≤ radius`，移动对象还需 `now-observed_at ≤ max_age`。若最多速度 v，坐标 age 为 a，则未知位置误差上界至少为 `v*a`；不能用更精确小数弥补陈旧信息。索引 lag 是 place version 提交到可查询 generation 的时间，召回边界同时受 cell 覆盖、候选上限、过滤和位置 age 影响。

分页游标要固定 cell 覆盖、索引 generation 与稳定排序键；否则对象跨 cell 或营业状态变化会导致重复和遗漏。密集 cell 必须按 place_id 去重，不能把来自旧、新 cell 的同一对象当作两个地点。

## 必然约束

[DEDUCED:nearby-places-spatial-index-only-produces-candidates] 空间 cell 索引只能产生候选集合，边界与近似编码要求对候选执行精确距离计算后才能判定半径资格。半径 100 米的圆跨过 cell 边界，若只查中心 cell 会漏掉 1 米外邻 cell 的地点；查完整 cell 又会带入角落超半径对象。

[DEDUCED:nearby-places-location-age-must-enter-eligibility] 移动地点或临时营业状态的时效必须进入资格判定，否则距离精确也会返回现实中已离开或已关闭的地点。一辆每秒 20 米的车，30 秒旧位置可偏离 600 米；所以 age 是业务资格，不只是监控标签。

[DEDUCED:nearby-places-filtering-requires-bounded-expansion] 属性过滤会耗尽初始空间候选，系统只能通过有预算的扩圈或接受召回不足，不能免费保证固定数量结果。初始 100 个候选仅 1% 营业，期望只剩一个；为凑 20 个必须扩大约二十倍候选，延迟与 I/O 同增。

## 从简单方案演进

小集合可全表扫描算距离并过滤，结果精确且更新立即可见。当扫描行 p95 超一万或 p99 超预算，增加固定网格索引。单层网格在城市候选过多、乡村 cell 为空；当 cell 候选 p99/中位数超过十倍，演进到层级 cell 并按密度选层级。

移动对象每次更新索引会抖动，采用权威位置日志加批量 cell 投影；当 index age p95 超 `max_age/2`，增加分区或缩小批次，而非把旧坐标称作实时。过滤后不足 K 时按 ring 逐级扩圈，每轮有候选/时间预算；当扩圈率超过百分之二十，可为高选择性属性建联合索引，但新增组合和写放大。

更快刷新成本约 `updates/sec * replicas * index writes`。未选预计算所有位置对距离，因状态二次方；封闭园区的少量固定点反而可以。也未默认按距离排序后无限扩圈；若产品允许“附近无结果”，受控不足比超时更诚实。

## 设计决定

选择层级 cell 候选、精确距离复核和当前资格过滤。更新按 place_id/version 写权威记录，投影消费单调版本，跨 cell 时旧 entry 以 tombstone 清理。查询固定 index generation，覆盖全部相交 cell，去重 place_id，按当前版本检查营业/租户/age，计算精确距离并稳定排序。

超时返回带 `coverage_radius` 和 index age 的部分结果，只在产品允许时启用；资格服务失败不回退到旧营业状态。重复更新幂等，旧版本拒绝。缓存键包含 cell 集、过滤摘要、generation；移动对象不做长 TTL。未选择数据库地理扫描作为长期主路径，但低 QPS 后台工具用它更简单且可作真值审计。

## 运行与演进

SLI 包括查询 p99、候选/结果比、边界真值 recall、location/index age、过滤淘汰率、扩圈次数、重复候选和 cell 热点。过载先降低 K 和最大扩圈，再禁用低价值过滤组合，不能扩大 max_age 伪装可用。位置数据最小化保留并按地区隔离。

故障时间线：10:00 对象 v21 从 cell A 移到 B；B 投影先写，A tombstone 延迟，查询两处都读到但以 id/version 去重；10:00:20 age 超阈值则即使 A 残留也不返回。演练边界圆覆盖遗漏和资格服务故障。灰度新 cell 级别时双索引，与全扫真值比较召回后切换。压测切换指标是候选/结果 p95 超五十、热点 cell CPU 七成；业务校准 max_age、允许不足 K 和半径误差。

## 面试考察本质

本题本质是：给定“半径与当前资格必须真实成立”的不变量，因为空间索引只是滞后近似且过滤选择率和移动位置未知，候选人必须推导候选生成、精确复核、位置时效和有界扩圈，再按召回、延迟与刷新成本决定索引粒度。

优秀回答会构造 cell 边界反例、写出 `v*a` 陈旧误差、区分静态 POI 与移动对象。常见误区是 geohash 前缀等于圆、承诺永远返回 K、只报平均密度。追问可进入极区/日期线、联合过滤、热点城市和隐私保留。20 分钟讲空间候选与精排，40 分钟补时效和扩圈，60 分钟讨论层级索引迁移、真值评估与多地域。
