# 全文搜索

## 表面题目

用户提交文本、过滤条件和排序方式，系统从持续变化的文档集合中返回相关且有权限查看的结果。成功不仅是“含有关键词”，还包括可解释的词法匹配、稳定分页、更新与删除在声明时限内可见，以及任何时刻不越过当前授权。品牌化搜索引擎名不能替代问题定义：索引是文档事实的派生投影，相关性分数也是某个语料统计快照下的计算结果。

## 反问与边界

先确认文档类型、字段与语言，关键词、短语、模糊匹配和语义检索哪些在范围内；排序按相关性、时间还是业务权重。问清索引写入到可搜的 SLO、ACL 撤销上限、查询 p99、K 与分页深度、更新/查询比例、单文档大小、高频词偏斜及租户隔离。是否要求 read-your-write，删除是搜索不可见还是物理擦除，跨地域能否接受旧索引，都需明确。本题专注倒排检索，不把爬虫获取或向量召回混为同一权威。

权威源是文档版本和授权服务；倒排 segment、词典、文档值、统计量与查询缓存均可重建。每个结果要能追溯 `doc_version,index_generation,acl_generation`。刷新越频繁，segment 越碎、合并和缓存失效越重。

## 客观模型

接口是 `Index(doc_id, version, fields)`、`Delete(doc_id, version)`、`Search(query, filters, snapshot, cursor)`。写入口只接受单调版本，转成不可变 segment；manifest 原子发布一组 segment 与删除位图。词 t 的 posting 长度为 `df(t)`，多词交集成本近似 `Σdf(t)`，而“the”之类高频词会把列表拉到文档总量级。

若写入速率为 W、刷新间隔为 R，每个小 segment 约含 `W*R` 文档，单位时间 segment 数约 `1/R`；减半 R 会近似加倍发布与合并压力。分页游标含 query_digest、index_generation、sort_after，不能只含 offset。索引 lag 定义为权威提交时间到包含该版本的可服务 generation 发布时间。

## 必然约束

[DEDUCED:full-text-search-index-is-lagging-derived-state] 倒排索引是权威文档的派生状态，刷新间隔内不可能同时获得零写放大、零查询协调与即时可见。提交后立刻搜索若不等待索引，就可能读旧；若查询同时查事务日志，则增加协调与去重；若每写即发布，则制造碎 segment。三者只能按 SLO 选择。

[DEDUCED:full-text-search-pagination-must-pin-query-snapshot] 游标必须固定索引与排序快照，否则 segment 发布和分数变化会让跨页结果重复或遗漏。反例：第一页后插入一个分数最高文档，offset=10 的第二页会重复原第十项；固定 generation 与 `sort_after(doc_id)` 才能声明一致页面。

[DEDUCED:full-text-search-acl-must-use-current-authorization] 索引中的 ACL 只能加速候选过滤，最终返回必须受当前授权代际裁决以阻断撤权后的旧 posting。用户在 10:00 被移出组，索引 10:05 才刷新；只看 posting 会泄露五分钟。当前 deny/授权代际在结果释放前检查，索引更新负责性能而非最终许可。

## 从简单方案演进

小数据先用数据库字段扫描，行为清楚且更新即时。当扫描行数 p95 超一万或查询 p99 超预算，构建单分片倒排索引。单 segment 更新困难，于是用不可变小 segment 加后台合并；当每查询触碰 segment p95 超二十，或 merge debt 超可用磁盘百分之二十，增加分层合并和流量节流。

文档量超过单节点存储或高频 posting 占满 CPU 后，按 doc_id 分片并广播查询，各分片返回局部 Top-K 再归并。它增加 fanout 尾延迟与全局统计偏差；当分片 p99 的最大值主导端到端，需副本择优、超时部分结果或按租户/语料路由。刷新 lag 超目标时缩短 R，但要监测 segment 数、合并 I/O 与缓存失效。

未选“按关键词分片”，因为多词查询需要跨词协调且热点词形成巨分片；单一词典型分析系统可能重新适合。也不默认实时查主库补齐，因为权限与排序去重复杂；只有 read-your-write 用户流量小且日志窗口有界时可叠加 session overlay。

## 设计决定

选择 doc_id 分片、不可变 segment 和原子 manifest。写按 `(doc_id,version)` 幂等，旧版本拒绝；删除写 tombstone 并立即推进 deny 代际，后台回收正文。查询先固定 generation，解析词项与过滤，分片并行取候选，协调器稳定归并；释放结果前批量检查当前 ACL generation。游标过期明确返回重启分页，而不静默换代。

单分片超时可在“允许部分结果”请求中标记 coverage 后返回，合规或完整导出请求则失败。索引发布失败继续服务旧代并暴露 lag；ACL 服务不可用时保护型查询 fail-closed。缓存键含规范化 query、租户、索引与授权代际。未选择就地更新 posting，因为崩溃中间态和回滚复杂。

## 运行与演进

SLI 包括查询 p50/p99、完整分片率、索引 lag、ACL revoke lag、零结果率、posting 解码量、segment 数、merge debt、缓存命中和每千查询 I/O。过载先禁用昂贵模糊扩展，再降低候选 K，允许的场景返回标记部分结果；授权检查绝不绕过。

故障时间线：10:00 文档 v8 提交，10:01 用户撤权并生成 ACL 44，旧索引仍含 v7；10:01:02 查询命中 posting，但释放门按 44 拒绝；10:03 manifest 发布 v8/tombstone。演练发布一半 coordinator 崩溃，读者只能看到完整旧或新 manifest。灰度分析器以新索引旁路比较词项覆盖、p99 和相关性，回滚只切指针。压测切换指标为 segment fanout p95 超二十、merge I/O 七成；业务校准刷新 lag、允许部分结果和 ACL 撤销上限。

## 面试考察本质

本题本质是：给定“结果来自一致查询快照且不得越过当前 ACL”的不变量，因为倒排索引与现实文档、权限只能异步同步，候选人必须推导刷新延迟、segment 写放大和查询协调之间的约束，再按写后可见、相关性和泄露风险选择快照与授权边界。

优秀回答会定义权威文档、manifest 与 ACL 的不同角色，量化 posting/segment 成本，并用 offset 反例解释分页。常见误区是“最终一致即可”覆盖撤权、每次更新全量重建、把分片局部 Top-K 当全局精确。追问可进入 BM25 统计、短语位置、深分页、删除回收与多租户。20 分钟讲索引结构，40 分钟补刷新和 ACL，60 分钟讨论分片召回、合并策略和灰度相关性评估。
