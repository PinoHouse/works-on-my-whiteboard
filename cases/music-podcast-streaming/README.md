# 音乐与播客流媒体

## 表面题目

设计音乐与播客流媒体，是让用户搜索或打开节目后快速开始连续音频，在移动网络波动时尽量不断声，跨设备同步有语义的播放进度，并允许获得授权的内容短期离线。成功不只是分片 200：播放器要从同一 asset generation 的时间轴连续解码，进度更新不能误把用户主动回听覆盖掉，离线字节只有在许可证权利窗口内可解密。

这题与视频点播共享分片，却有不同主导体验：音频用户可能锁屏连续数小时，中断比瞬时清晰度下降更显著；播客需要精确续播，音乐曲目还涉及队列、跨曲预取、无缝衔接和版权地域。本文不设计推荐排序或版税结算，只保留可审计播放事件接口；目录元数据、媒体字节、进度和权利是四个不同事实域。

## 反问与边界

先问音乐/播客比例、平均时长、启动与 stall SLO、码率档、峰值并发、离线时长、家庭账户、地域版权、跨设备接力及进度冲突语义。用户拖回 30 秒是新意图还是旧进度，多个设备同时播放是否允许，撤权要求立即还是在许可证到期时生效。下载成功应区分字节完整、license 可用和整个队列可离线，避免“已下载”却开播失败。

设并发 `V`、平均码率 `r`，出口约 `V×r`；每首存 `k` 个码率，媒体存储放大约 `Σri/rmaster`。离线下载 `D` 台设备、每台 `L` 小时，最大客户端持有字节约 `D×L×r/8`，但缓存淘汰和重复曲目会改变实际值。音频分片 `s` 秒时每小时约 `3600/s` 次请求；太短增加无线唤醒和请求开销，太长则拖慢 seek、码率切换与撤销窗口。

## 客观模型

实体是 `AudioAssetGeneration`、`RenditionTimeline`、`CatalogRelease`、`PlaybackQueueRevision`、`PlaybackSession(session_id,device_id,asset_generation,position,seq)`、`ProgressRecord`、`OfflinePackage` 和 `License(epoch,rights,expires,device_binding)`。媒体 segment 不可变且各 rendition 时间戳对齐；目录逻辑项指向 asset generation。在线会话获得绑定 generation 和 policy 的短令牌，离线包存密文字节，只有设备绑定 license 可解。

播放器缓冲变化为 `buffer(t+Δ)=buffer(t)+downloaded_bytes/r-played_time` 的等价时间量；当预测吞吐不能覆盖当前码率和安全余量时，应在缓冲耗尽前切到同时间轴低档。进度事件携带 session id、单调 session sequence、asset generation、position、reason(play/seek/complete) 和观察到的云版本。服务对同会话重试幂等，对跨会话并发不以服务器到达时间盲目覆盖。

进度的默认合并可定义：正常播放只推进已知连续进度；显式 seek-back 记录为新用户意图并带因果版本；另一设备在未观察此意图时上传更大进度，形成需策略选择的并发，而非自动证明“更大更新”。音乐短曲可偏向当前活动会话，播客长节目可提示冲突。license 权威拥有 rights epoch；设备离线时只能按已签发到期时间执行。

## 必然约束

[DEDUCED:music-podcast-streaming-continuity-outranks-peak-bitrate] 用户以 256 kb/s 播放，网络未来十秒只能稳定 128 kb/s，若坚持最高档，缓冲耗尽必然静音；同一时间轴的 96 kb/s 档虽降低音质，却保持连续。对于声明“不中断优先”的产品，ABR 应在预测耗尽前降档，而非等失败。这个结论不适用于无损下载完整后播放，但在线弱网下最高码率与连续性不可同时保证。

[DEDUCED:music-podcast-streaming-offline-availability-conflicts-with-instant-revocation] 设备在飞机模式持有七天许可证，服务在第二天撤销版权，设备没有通信渠道获知。要么它继续到许可证到期，撤权最多延迟五天；要么要求联网确认，便失去真正离线播放。短 license、限制下载窗口和高风险内容不允许离线只能界定暴露，不能消除信息下界。已解密输出也无法远程收回。

[DEDUCED:music-podcast-streaming-progress-merge-needs-session-causality] 设备 A 听到 50 分钟后主动回到 20 分钟，设备 B 离线时从 40 听到 45，随后 B 的 45 最后到达。若取最大值会抹掉 A 的回听意图，若最后写胜则网络顺序决定。事件必须标 session、seq、显式 seek 与 observed version；产品再按活动会话或提示决定，而不能把任一标量规则冒充普遍正确。

## 从简单方案演进

最简单基线是单码率整文件下载并本地记进度，适合短音频与单设备。点击首播 `p95` 超标或平均只听前 30% 却下载整集时切成分片流式；新增请求与时间轴。移动网络下 stall 会话超过待校准 0.5%，且吞吐分布跨多个档位时加入对齐 ABR；若用户主要在 Wi-Fi 且音质比连续更重要，单高码率重新更优。

当跨设备用户占比超过 10% 或“找不到上次位置”投诉达到阈值，进度从简单最大值演进为 session 事件与因果版本；复杂度是冲突策略和存储。离线请求增长后发行密文 package 与短期 license；若 license refresh 失败导致开播失败超过目标，延长窗口会改善可用却扩大撤权暴露，需要版权方校准。两组待演练指标是缓冲低于多少秒降档最有效，以及 license 为 24/72 小时时离线成功与撤权风险曲线。

未选无限期离线密钥，因为无法撤权；只有用户永久购买且条款允许时重新变优。未选每个进度事件全局强同步，因为离线设备不能参与且每秒写入成本高；若是医疗/培训合规要求精确观看证明且设备始终在线，可用更强协调。目录更新采用新 asset generation，不覆盖在播字节。

## 设计决定

发布流程产出对齐 rendition manifest 并原子切目录 generation。播放网关验证当前地域/订阅，签发短会话；播放器预取下一片与队列下一曲的首片，依据吞吐和缓冲降档，升档更保守。segment 重试按 digest 幂等。进度本地持久化，按重要事件与周期批量上传；服务以 session seq 去重，并保留显式 seek 和并发版本供产品规则合并。

离线下载先取得绑定 device、asset、rights epoch 和 expires 的 license，再下载并摘要验证密文包；开播在可信时钟与回滚防护内检查。过载从低到高：暂停封面/歌词预取；停止升档和无损；缩短跨曲预取；延后进度非关键心跳但保留 seek/complete；限制新离线下载；最后排队新播放，已有音频会话优先。绝不以错曲、错 generation 或越权 stale 兜底。

反选让 CDN 签名 URL 本身承担离线授权，因为 URL 被复制后缺少设备绑定、权利 epoch 和离线状态机；公开免费播客可直接长缓存时它重新足够。反选每秒同步进度，因为无线唤醒与写放大大于收益；若实时多人共听需要亚秒共享位置，应另建协同会话而不是复用个人进度。

## 运行与演进

SLI 包括音频启动、stall 次数/时长、连续播放小时、码率切换、segment miss、预取浪费、进度冲突/回退、跨设备接力成功、离线包完整、license refresh/过期拒绝及撤权暴露窗口。切换阈值一：缓冲低于不同秒数时的 stall 概率与降档收益；阈值二：离线许可时长对成功率和撤权最大延迟的影响。按音乐、播客、网络和设备分层，平均值没有决策意义。

故障时间线：0 分钟设备 A 会话 7 听到 50 并显式 seek 20，上传 observed version 12；同时离线 B 基于 version 11 从 40 到 45；B 后到时服务标并发，不覆盖 A 意图。另一演练中 rights epoch 9 的 48 小时 license 已发，12 小时后控制面升 10；在线会话立即拒绝旧 epoch，离线设备最多播放到 expiry，指标明确记录剩余暴露而非声称即时撤销。

编码升级双产出新 asset generation，检查时间轴长度、响度、gapless 元数据和摘要后灰度目录；在播会话固定旧 generation，回滚只切目录。设备密钥轮换保留短兼容期，丢失设备加入 deny list 后停止续签。多租户按出口、离线字节和 license QPS 隔离，播放事件去标识化并设置保留期，不能为了续播无限保存详细收听历史。

## 面试考察本质

本题本质是：给定“音频必须在同一 asset generation 上连续，离线字节只能在有效权利窗口内播放，进度要保留用户意图”，因为未来网络、离线期间撤权和另一设备动作无法立即获知，而带宽、电量和授权窗口有限，候选人应推导缓冲 ABR、有限 license 和带因果的进度合并，并决定音质、连续性、离线可用与撤权风险的优先级。

优秀回答会把媒体、目录、进度与权利拆开，算 `V×r` 和离线持有量，给出 A 回听/B 迟到及飞机模式撤权反例，说明何时降档。常见误区是把它当低码率视频点播、进度永远取 max、签名 URL 永久离线、宣称撤权能收回已下载字节，或过载先中断已有播放。追问可落到 gapless、播客章节、家庭账户、购买内容和可信时钟。

二十分钟完成连续性 SLO、媒体版本、缓冲与 license；四十分钟加入进度并发、离线、降级和故障；六十分钟再谈版权地域、设备密钥、隐私和编码迁移。答案的独特性在于能解释“不断声”“继续上次位置”和“可离线但可撤权”各自需要何种事实，而不是再画一次通用 CDN。
